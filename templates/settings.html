Authentication Mode

Configure how users authenticate to TimeKpr WebUI

Local: Standard admin login with username and password.
External: Authentication delegated to reverse proxy (Caddy, nginx, Traefik, etc).

Synchronization

Control how often shared time pools are reconciled across hosts

Minimum drift between the desired and current time left before a correction is pushed. Lower values keep the pool more precise but generate more updates (and desktop notifications). Default: 300 s (5 min).
How often to evaluate a group when time has been used today. Default: 120 s (2 min).
How often to evaluate a group when no time has been used today. Default: 180 s (3 min).
When enabled, the host whose time is actively decreasing will not receive a correction push — timekpr already accounts time locally on that machine. This eliminates notifications on the screen the child is using.

Security Tips

  • Use a strong password with at least 8 characters
  • Include uppercase, lowercase, numbers, and symbols
  • Ensure all remote systems use the same password for the timekpr-remote user
